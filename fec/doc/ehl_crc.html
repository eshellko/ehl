ehl_crc - Cyclic Redundancy Check

Модуль подсчета циклической контрольной суммы.

    _______________________
   |                       |
---| data_in   crc_out     |---
---| crc_in    crc_correct |---
   |_______________________|

Описание

  • Блок генерации/проверки контрольной суммы.
  • Параметризуемое значение полинома.
  • Ширина шины данных определяется параметром DATA_WIDTH.

    • Параметры

  • DATA_WIDTH – количество бит во входных данных. Допустимые значения от 1 до 1024 (значение по умолчанию 4).
  • POLY_WIDTH – количество бит в контрольной сумме. Допустимые значения от 1 до 1024 (значение по умолчанию 1024).
  • INIT_VALUE — начальное значение полинома (0 — все нули, 1 — все единицы (например, Ethernet)).
  • BIT_ORDER – порядок обработки бит в data_in. 0 – LSB, 1 – MSB (значение по умолчанию 0).
  • XOR_MSB — значение старшего бита полинома (1 для Ethernet).
  • POLY_COEF – генерирующий полином (значение по умолчанию 0x04C11DB7).

    • Порты

    Название Разрядность Направление Описание
    data_in DATA_WIDTH Вход Входные данные
    crc_in POLY_WIDTH Вход Текущее значение CRC
    crc_out POLY_WIDTH Выход Следующее значение CRC
    crc_correct 1 Выход Результат сравнения с ожидаемым остатком

    Пример функционирования

    Используется для генерации / проверки контрольной суммы. При генерации выход crc_correct следует оставить неподключенным.

    При проверке контрольной суммы пакетов данных USB (CRC16) следует задать параметры POLY_WIDTH = 16, POLY_COEF = 16’h8005. Остальные параметры зависят от реализации схемы подачи данных на ehl_crc.

    При проверке контрольной суммы пакетов данных Ethernet (CRC32) следует задать параметры POLY_WIDTH = 32, POLY_COEF = 32’h04C11DB7. Остальные параметры зависят от реализации схемы подачи данных на ehl_crc.

    Может применяться для проверки целостности содержимого области памяти перед началом исполнения харнящегося в ней кода.
