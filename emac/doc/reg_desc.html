Описание регистров ehl_emac

TX_CTRL [0x0]: Регистр управления передатчиком.

Биты Название Описание Доступ Начальное значение
31:1 - Резерв R 0
0 TE Transmit Enable. Разрешение работы передатчика:
  • 0 - запрещено;
  • 1 - разрешено.
    		•  R/W 0

    RX_CTRL [0x100]: Регистр управления приемником.

    Биты Название Описание Доступ Начальное значение
    31:1 - Резерв R 0
    0 RE Receive Enable. Разрешение работы приемника:
  • 0 - запрещено;
  • 1 - разрешено.
    		•  R/W 0

    MAC0 [0x200]: Младшие 32 бита MAC адреса контроллера.

    MAC1 [0x204]: Старшие 16 бит MAC адреса контроллера.

    Биты Название Описание Доступ Начальное значение
    63:48 - Резерв R 0
    47:39 MAC Старшие биты MAC адреса. R/W 0
    40 - Резерв R 0
    38:0 - Младшие биты MAC адреса. R/W 0

    MINMAX [0x210]: Управление мин/макс размерами кадров.

    Биты Название Описание Доступ Начальное значение
    31 - Резерв R 0
    30:16 MAXL Maximum Length. Значение в байтах максимального размера кадра. При передаче ограничивает размер данных, записываемых в буфер. Включает в себя значение полей Destination Address, Source Address (опционально при записи в буфер), Length/Type, Data (опционально при записи в буфер) и CRC (не передается в буфер). При приеме ограничивает размер принятого кадра, включая в себя поля Source Address, Destination Address, Length/Type, Data/PAD, CRC. Значение MaxLR:
  • не должно превышать 4096 байт;
  • должно быть больше 256 байт;
  • должно быть больше значения MinLR.
    • Пользователь несет ответственность за запись корректных значений в регистр MaxLR. Размер максимально допустимого кадра. Определяет максимальную длину кадра в байтах. Стандартное значение 1518 байт.     		R/W 0
    15:10 - Резерв R 0
    9:0 MINL Minimum Length. Значение в байтах минимального размера кадра. При передаче данные, записываемые в буфер, должны содержать не менее 6 байт Destination Address + 6 байт Source Address (опционально) + 2 байт Length/Type. Оставшееся значение до MinLR заполняется полями PAD и CRC при передаче. При приеме ограничивает размер принятого кадра, включая Source Address, Destination Address, Length/Type, Data/Pad, CRC. Значение MinLR:
  • не должно превышать 512 байт.
  • должно быть меньше MaxLR.
    • Пользователь несет ответственность за запись корректных значений в регистр MinLR. Размер минимально допустимого кадра. Определяет минимальную длину кадра в байтах. Включает в себя поля DA (6 байт), SA (6 байт), LT (2 байта), DATA, CRC (4 байта). Стандартное значение 64 байта.     		R/W 0
